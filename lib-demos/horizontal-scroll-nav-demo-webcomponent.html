Horizontal Scroll Navigation - Web Component Mode

Uses <horscroller-viewport /> and .horscroller-section classes.

⚔ Warriors & Fighters

This section is for warriors. The horizontal navigation bar will create a button for this section based on the h2 title. As you scroll, the button fills with progress.

Notice how the navigation bar appears below the header and stays sticky at the top as you scroll. The button for this section will show your progress through it.

🎯 Strategists & Commanders

This is the strategists section. Each section can have different content and different heights. The navigation system normalizes everything - a 200px button represents an 8000px section proportionally.

Scroll through this section and watch how the button fills. The viewport indicator (blue/purple) shows exactly where your current viewport is within the section.

💰 Merchants & Logistics

The merchants section. Notice how clicking a button in the navigation bar smoothly scrolls you to that section. The navigation updates correctly once you arrive.

The horizontal nav bar also scrolls to keep the active section's button visible, moving proportionally as you scroll through the section.

🏥 Support Roles

Support roles section. The plugin automatically extracts the section title from the h2 element to use as the button label. You can customize this with the getSectionLabel option.

All sections are automatically detected within the horscroller-viewport element.

🎮 Gaming Skills Transfer

This is a longer section to demonstrate how the normalized scrolling works. Even if this section is 8000px tall and the button is only 200px wide, the scroll position maps correctly.

Scroll through this section and watch the button fill. The viewport indicator moves across the button showing your current position. The horizontal nav bar scrolls to keep the button visible.

This demonstrates the "normalized" binding - the section size doesn't matter, everything is proportionally mapped between the section height and button width.
